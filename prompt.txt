## STEP 3 — Conversion rules (STRICT — follow exactly)

### 3.1 — CSS extraction
- Extract ALL styles from every Stitch HTML file (inline styles, <style> tags, imported classes)
- Merge them into a SINGLE file: `public/css/style.css`
- Deduplicate classes — if the same class appears in multiple exports, keep it once
- Organize the CSS file with comments by section:
  /* === GLOBAL === */
  /* === LAYOUT / HEADER / FOOTER === */
  /* === AUTH === */
  /* === GAMES === */
  /* === RESERVATIONS === */
  /* === SESSIONS === */

### 3.2 — HTML → PHP conversion
For each target PHP file:
- Copy the HTML structure from the Stitch export EXACTLY — do not change layout, classes, colors or spacing
- Remove the <html>, <head>, <body> wrapper tags (these are partials, not full pages)
- Remove any <style> or <link> tags that were inside the export (CSS is now in style.css)
- Replace every hardcoded text value (names, dates, numbers, game titles) with a PHP echo placeholder:
    "Catan"              → <?= htmlspecialchars($game['name']) ?>
    "Stratégie"          → <?= htmlspecialchars($game['category']) ?>
    "14:00"              → <?= $reservation['reserved_at'] ?>
    "Karim Benali"       → <?= htmlspecialchars($user['name']) ?>
  Keep the placeholder variable names consistent with the project models:
    Games    → $game, $games
    Reservations → $reservation, $reservations
    Sessions → $session, $sessions
    Users    → $user
- Replace hardcoded lists/loops with PHP foreach:
    A grid of game cards → <?php foreach ($games as $game): ?> ... <?php endforeach; ?>
    A table of reservations → <?php foreach ($reservations as $reservation): ?> ... <?php endforeach; ?>
- For status badges with conditional colors, use:
    <?php if ($game['status'] === 'available'): ?> class="badge badge-green" <?php else: ?> class="badge badge-red" <?php endif; ?>
- Keep all hardcoded example data for components that are NOT dynamic 
  (e.g. the filter buttons Stratégie/Ambiance/Famille/Experts stay hardcoded)

### 3.3 — header.php and footer.php
- header.php must include at the top:
    <link rel="stylesheet" href="/css/style.css">
- All other view files must include header and footer using:
    <?php require __DIR__ . '/../layouts/header.php'; ?>
    (at the very top, before any HTML)
    <?php require __DIR__ . '/../layouts/footer.php'; ?>
    (at the very bottom, after all HTML)
- Exception: login.php and register.php should NOT include header/footer 
  (they are standalone pages), but they must link the CSS directly with:
    <link rel="stylesheet" href="/css/style.css">

### 3.4 — Forms
- All <form> tags must have the correct method and action:
    login form      → method="POST" action="/login"
    register form   → method="POST" action="/register"
    game create     → method="POST" action="/games"
    game edit       → method="POST" action="/games/<?= $game['id'] ?>/update"
    game delete btn → method="POST" action="/games/<?= $game['id'] ?>/delete"
    reservation create → method="POST" action="/reservations"
    reservation status → method="POST" action="/reservations/<?= $reservation['id'] ?>/status"
    session start   → method="POST" action="/sessions"
    session end btn → method="POST" action="/sessions/<?= $session['id'] ?>/end"
- Add <?= $_SESSION['csrf_token'] ?? '' ?> as a hidden input if a token field exists in the design

### 3.5 — Error display
- In forms (login, register, games/create, games/edit, reservations/create), add this block 
  just above the first form field:
    <?php if (!empty($errors)): ?>
      <div class="error-box">
        <?php foreach ($errors as $error): ?>
          <p><?= htmlspecialchars($error) ?></p>
        <?php endforeach; ?>
      </div>
    <?php endif; ?>
  Style .error-box in style.css as a red-bordered box matching the app's dark theme.
